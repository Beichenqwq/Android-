第0部分 Android开发环境的设置与准备

俗话说的好，工欲善其事必先利其器。我们首先来学习开发Android应用之前的环境搭建工作。

在以前，Android应用的开发工具大多采用Eclipse，现在我们依然可以使用该IDE工具作为我们的开发工具，但是，Android官方网站推荐使用 Android Studio作为IDE开发工具。Android Studio 是基于 IntelliJ IDEA 构建的官方 Android 应用集成开发环境 (IDE)。

1. Android Studio的安装、配置

接下来，我们一起来安装和配置Android Studio。

1.1. Android Studio的安装

首先我们来到 Android Studio 的官方下载网站，下载Android Studio的最新安装文件。或者到：http://www.android-studio.org/下载安装包进行安装。找到下载文件，双击打开，安装过程基本上一路next即可。

注意：在windows系统中，在安装Android Studio的过程，是可以更改它的安装路径的，但需要注意的是，在它的安装路径中不要出现中文字符，以免未来出现一些因为字符原因导致的系统错误！

在安装完成以后，因为Android Studio安装程序目前不会自动在你的桌面创建Android Studio启动快捷方式，这时，为了方便，你可以找到它的安装目录（需要你在安装Android Studio的时候记下），一般是在"C:\Program Files\Android\Android Studio\bin\"中，找到studio64.exe，右击，选择“发送到 > 桌面快捷方式”。

详细过程大家可以参考视频。在制作视频以来，Android Studio又经过了Google的不断地升级完善，它的安装及操作方式都会有些许变化，但基本流程没有大的变化，读者不应生搬硬套，应在理解流程含义的基础上灵活掌握。

1.2. Android SDK的配置与更新

在安装完成以后，我们还可以设置Android SDK。Android SDK为Android的开发工具包，在做Android应用开发的时候，必须有SDK的支持。

至于Android SDK的获取，在安装Android Studio的过程中，它可以从网络中下载。但是如果你以前有下载过，可以在安装过Android Studio以后，通过打开SDK Manager进行设置，修改为你以前的SDK路径。如图为首次打开Android Studio的欢迎界面，按照图中的数字顺序选择执行，进行配置。

注意：SDK所在的磁盘路径名称中千万不要含有中文及中文符号

另外，Android SDK也需要经常进行更新，方法也是打开SDK Manager，使用SDK Manager对SDK进行更新。

如果是在已经打开有项目（project）的情况下，需要打开SDK Manager，点击菜单 Tools > SDK Manager 或点击工具栏中的 SDK Manager图标。

已安装的软件包如有更新，其旁边的复选框中会显示短划线。

  • 要更新项目或安装新项目，请点击复选框，其将显示为对号符号。
  • 如需卸载软件包，请点击以清除复选框。
  • 待下载的更新在左侧列中以下载图标表示。待执行的删除项左侧以❌表示。

1.3. Android Studio的更新

安装 Android Studio 后，还可以通过自动更新或手动更新 Android Studio IDE 以及Android SDK。

若有可用的 IDE 更新，Android Studio 将通过小气泡对话框进行通知，也可以点击 Help > Check for Update（在 Mac 中，点击 Android Studio > Check for Updates）手动检查更新。

2. 虚拟机的安装

我们在写Android应用程序的时候，需要测试运行它，而Android应用不能直接在我们的IDE环境中运行。那怎么办呢？一种方法，采用真机调试，打开我们自己手机的开发者模式，通过USB线连接到计算机；另一种方法就是采用虚拟机的方式。我们可以采用Android官方提供的虚拟机，但是官方的虚拟机的运行速度比较慢，我们可以寻求使用第三方的虚拟机。

2.1. Genymotion

这里我们先介绍使用Genymotion。

2.1.1. Genymotion安装与配置

打开Genymotion的官方网站，Genymotion有两个版本：Cloud version和Desktop version，选择其中一种版本。如果是作为个人使用，建议使用Desktop version。

在下载Desktop version时，有两种选择，一种是集成VirtualBox的版本，一种是没有集成VirtualBox的版本，VirtualBox为Genymotion提供虚拟机支持，Genymotion必须使用VirtualBox方可工作，若下载没有集成VirtualBox的版本，前提是你的计算机上已经安装有VirtualBox。根据自己的情况，选择下载。

安装完成以后，第一次启动时，需要输入账号，如果已有账号，将账号输入即可，否则选择创建新的账号。在输入账号过程中，选择license时，选择personal use，如图：

安装完成以后，记得设置一下Genymotion的ADB，将ADB设置为Android SDK，如果这项不设置好，在Android Studio中就无法正常连接虚拟机。打开Genymotion > Settings > ADB，如下图：

在该图的Android SDK后面可以通过设置Android SDK的磁盘位置。

在正式进入Genymotion以后，需要添加虚拟机。点击界面右上角的＋，打开如图所示的界面：

在这个界面里，按照Type、Device及Android API选择所需创建的虚拟机，点击NEXT，进入下一个界面：

在此界面中对虚拟机的参数进行微调，点击INSTALL，进行安装。

安装完成以后，选择需要启动的虚拟机后面的三个点，选择Start，正常情况下，虚拟手机就会启动。如图：

最后，若需要虚拟机正常使用，则必须从BIOS中开启VT(Virtual Technology)，如何开启，请按照你的计算机的使用说明，或者百度一下。

以上步骤完成以后，我们必须让Android Studio能够使用Genymotion，我们需要给Android Studio安装Genymotion插件

打开Android Studio，依次 File > Settings，在打开的settings界面里找到plugins设置项，如图：

在搜索栏里输入genymotion关键字，点击install安装。

3. Android Studio的使用

这部分，大家可以作为使用参考手册来使用。

3.1. 界面

Android Studio 主窗口由下图标注的几个逻辑区域组成。

  1. 工具栏提供执行各种操作的工具，包括运行应用和启动 Android 工具。
  2. 导航栏可帮助您在项目中导航，以及打开文件进行编辑。此区域提供 Project 窗口所示结构的精简视图。
  3. 编辑器窗口是创建和修改代码的区域。 编辑器可能因当前文件类型的不同而有所差异。 例如，在查看布局文件时，编辑器显示布局编辑器。
  4. 工具窗口栏在 IDE 窗口外部运行，并且包含可用于展开或折叠各个工具窗口的按钮。
  5. 工具窗口提供对特定任务的访问，例如项目管理、搜索和版本控制等。您可以展开和折叠这些窗口。
  6. 状态栏显示项目和 IDE 本身的状态以及任何警告或消息。

您可以通过隐藏或移动工具栏和工具窗口调整主窗口，以便留出更多屏幕空间。 您还可以使用键盘快捷键访问大多数 IDE 功能。

您可以随时通过按两下 Shift 键或点击 Android Studio 窗口右上角的放大镜搜索源代码、数据库、操作和用户界面的元素等。此功能非常实用，例如在您忘记如何触发特定 IDE 操作时，可以利用此功能进行查找。

3.3. 样式和格式化

在您编辑时，Android Studio 将自动应用代码样式设置中指定的格式设置和样式。 您可以通过编程语言自定义代码样式设置，包括指定选项卡和缩进、空格、换行、花括号以及空白行的约定。要自定义代码样式设置，请点击 File > Settings > Editor > Code Style（在 Mac 上，点击 Android Studio > Preferences > Editor > Code Style）。

虽然 IDE 会在您工作时自动应用格式化，但您也可以通过按 Control+Alt+L（在 Mac 上，按 Opt+Command+L）显式调用重新格式化代码操作，或按 Control+Alt+I（在 Mac 上，按 Alt+Option+I）自动缩进所有行。

4. Android Gradle 插件

Android Studio 编译系统以 Gradle 为基础，并且 Android Gradle 插件添加了几项专用于编译 Android 应用的功能。虽然 Android Gradle 插件通常会与 Android Studio 的更新步调保持一致，但插件（以及 Gradle 系统的其余部分）可独立于 Android Studio 运行并单独更新。

4.1. 更新 Android Gradle 插件

在更新 Android Studio 时，您可能会收到将 Android Gradle 插件自动更新为最新可用版本的提示。您可以选择接受该更新，也可以根据项目的编译要求手动指定版本。

您可以在 Android Studio 的 File > Project Structure > Project 菜单中指定插件版本，如图：

4.2. 更新 Gradle

在更新 Android Studio 时，您可能会收到一并将 Gradle 更新为最新可用版本的提示。您可以选择接受该更新，也可以根据项目的编译要求手动指定版本。

下表列出了各个 Android Gradle 插件版本所需的 Gradle 版本。要获得最佳性能，您应该使用 Gradle 和插件这两者的最新版本。

您可以在 Android Studio 的 File > Project Structure > Project 菜单中指定 Gradle 版本，也可以通过在 gradle/wrapper/gradle-wrapper.properties 文件中修改 Gradle 分发引用来指定。以下示例在 gradle-wrapper.properties 文件中将 Gradle 的版本设置为 7.4.2。

4.3. Gradle 对依赖库的管理

在 settings.gradle 文件中，有对依赖库的指定，如下：

pluginManagement {
  repositories {
    gradlePluginPortal()
    google()
    mavenCentral()
  }
}

dependencyResolutionManagement {
  repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
  repositories {
    google()
    mavenCentral()
  }
}
rootProject.name = 'GradleManagedDeviceTestingNew'
include ':app'

这里google()所确定的源，如果无法访问的话，我们可以将maven仓库指定到阿里云的maven代理，阿里云的帮助文档为：阿里公共代理库。按照文档所述，我们将 settings.gradle 文件中的repositories内容替换为：

pluginManagement {
  repositories {
    gradlePluginPortal()
    maven { url 'https://maven.aliyun.com/repository/google' }
    //google()
    mavenCentral()
  }
}

dependencyResolutionManagement {
  repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
  repositories {
    maven { url 'https://maven.aliyun.com/repository/google' }
    //google()
    mavenCentral()
  }
}
rootProject.name = 'GradleManagedDeviceTestingNew'
include ':app'
